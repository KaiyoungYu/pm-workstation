# 模块设计文档模板（Module PRD Template）

> **用途**：本文件为 `knowledge/02-requirements/REQ_*/PRD/modules/` 和 `knowledge/02-requirements/REQ_*/PRD/components/` 下各模块/组件设计文档的标准模板。Agent 在编写或评审模块 PRD 时，应以此模板为基准。
>
> **核心原则**：模块 PRD 是页面 PRD 的延伸，聚焦**单一模块/组件**的界面结构、配置项、交互逻辑和数据说明。不重复描述页面级信息（用户旅程、导航、信息架构）。
>
> **与页面 PRD 的分工**：页面 PRD 回答「用户在哪、怎么到这儿、怎么离开」；模块 PRD 回答「这个区域内有什么、怎么配、点了会怎样」。

---

## 模块级可见元素记录规范

当 PRD 描述一个具体模块/组件时，必须穷尽记录模块内所有**可见元素**的文字、交互与状态。以下 checklist 用于自审：

### 必须记录的文字内容
- [ ] **所有占位文本（placeholder）**：输入框、搜索框、文本域的 placeholder 必须逐字记录
- [ ] **所有按钮/操作文字**：包括主按钮、次按钮、文字链接、图标按钮的 tooltip 文字
- [ ] **所有标签/标题文字**：区域标题、字段标签、列标题、Tab 标签
- [ ] **所有空态/默认态文字**：无数据提示、默认选中态提示、加载提示
- [ ] **所有错误/校验提示文字**：报错信息、确认提示、二次确认文案
- [ ] **所有状态标签文字**：如"等待中"、"清洗中"、"已完成"等状态映射

### 必须记录的交互细节
- [ ] **所有可点击元素的触发结果**：点击后发生什么（打开弹窗/抽屉、展开折叠、触发请求）
- [ ] **所有输入组件的约束**：maxLength、rows、showCount、allowClear、是否禁用等
- [ ] **所有选择组件的选项列表**：Select、Radio、Checkbox 的选项文字与默认值
- [ ] **所有开关/滑块的默认状态**：默认开/关，切换后的即时效果
- [ ] **所有 Hover/Focus 效果**：鼠标悬停时的视觉反馈（阴影、颜色变化、图标显示）
- [ ] **所有键盘交互**：Enter 提交、Esc 关闭、Tab 切换焦点等

### 必须记录的状态与边界
- [ ] **所有加载态**：加载中的展示方式（骨架屏、loading 图标、进度条、区域锁定）
- [ ] **所有空态**：无数据时的模块展示
- [ ] **所有错误态**：网络错误、权限不足、数据异常时的模块展示
- [ ] **所有二次确认场景**：删除、重置、发布等危险操作的确认流程
- [ ] **所有联动关系**：A 配置项变化如何影响 B 配置项（级联选择、联动禁用、动态显隐）

### 样式记录原则
- **有则写，无则省**：如果某个元素使用了特殊样式（非系统默认），需说明；如果使用的是组件库默认样式，不重复描述
- **动画/过渡**：如果有特殊的过渡动画（如展开折叠、抽屉滑入），需说明持续时间和缓动函数
