# PRD 模板示例（Product Requirements Document）

> **用途**：本文件为 `knowledge/02-requirements/REQ_*/PRD/` 下各设计文档的标准模板。Agent 在编写或评审 PRD 时，应以此模板为基准，确保设计在 BRD 设定的需求范围内展开。
>
> **核心原则**：PRD 回答 "What — 做什么？"，在 BRD 已确认的需求范围内设计功能，关心功能模块的划分、子功能之间的衔接、交互流程和原型。**不涉及接口路径、参数命名、数据类型、校验规则等技术细节**（这些在 SRD 中定义）。

---

## 页面级 PRD 可见元素记录规范

当 PRD 描述一个具体页面（而非抽象模块）时，必须穷尽记录页面上所有**可见元素**的文字、交互与状态。以下 checklist 用于自审：

### 必须记录的文字内容
- [ ] **所有占位文本（placeholder）**：输入框、搜索框、文本域的 placeholder 必须逐字记录
- [ ] **所有按钮文字**：包括主按钮、次按钮、文字链接、图标按钮的 tooltip 文字
- [ ] **所有标签/标题文字**：区域标题、字段标签、列标题、Tab 标签、面包屑文字
- [ ] **所有空态/默认态文字**：无数据提示、默认选中态提示、加载提示
- [ ] **所有错误/校验提示文字**：报错信息、确认提示、二次确认文案
- [ ] **所有状态标签文字**：如"等待中"、"清洗中"、"已完成"等状态映射

### 必须记录的交互细节
- [ ] **所有可点击元素的触发结果**：点击后发生什么（打开弹窗/抽屉、切换 Tab、跳转页面、触发请求）
- [ ] **所有输入组件的约束**：maxLength、rows、showCount、allowClear、是否禁用等
- [ ] **所有选择组件的选项列表**：Select、Radio、Checkbox 的选项文字与默认值
- [ ] **所有开关/滑块的默认状态**：默认开/关，切换后的即时效果
- [ ] **所有 Hover/Focus 效果**：鼠标悬停时的视觉反馈（阴影、颜色变化、图标显示）
- [ ] **所有键盘交互**：Enter 提交、Esc 关闭、Tab 切换焦点等

### 必须记录的状态与边界
- [ ] **所有加载态**：加载中的展示方式（骨架屏、loading 图标、进度条）
- [ ] **所有空态**：无数据时的页面展示
- [ ] **所有错误态**：网络错误、权限不足、数据异常时的页面展示
- [ ] **所有二次确认场景**：删除、发布、重置等危险操作的确认流程
- [ ] **所有联动关系**：A 组件变化如何影响 B 组件（搜索过滤、级联选择、联动禁用）

### 样式记录原则
- **有则写，无则省**：如果某个元素使用了特殊样式（非系统默认），需说明；如果使用的是组件库默认样式，不重复描述
- **响应式断点**：如果页面在移动端/平板/桌面端有不同布局，需说明断点和变化规则
- **动画/过渡**：如果有特殊的过渡动画（如抽屉滑入、模态弹窗淡入），需说明持续时间和缓动函数
